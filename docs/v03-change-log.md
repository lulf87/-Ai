# V0.3 改造说明

本次改造把现有 V0.2 的 evidence-first Demo 升级为“注册资料发补风险驾驶舱”雏形，并保留真实 LLM 默认、法规附件 SHA 校验、未校验法规不得作为规则依据、AI 候选需人工确认等安全边界。

## 已并入能力

- 新增老板驾驶舱 API 和前端数据带，展示申报准备度、红黄绿风险、重大断点、责任人分布和下一步动作。
- 新增主数据一致性矩阵 API，横向比较产品名称、型号规格、结构组成、适用范围、软件版本、使用寿命和检验/代表型号。
- 规则引擎新增资料完整性、软件/网络安全/AI 专项资料缺口、PTQ-检测报告覆盖矩阵、多型号覆盖逻辑、条款级义务库证据缺口等检查。
- 风险发现新增 `owner`、`workload`、`category` 字段，用于把发现转成整改闭环。
- Word 报告第一页改为老板摘要，包含项目总体风险、申报准备度、重大申报断点、责任人分布和下一步动作。
- 新增 `knowledge_base/regulatory_obligations_v03.json` 和 `config/risk_rules_v03.json`，作为义务库和配置化规则雏形。

## 未作为最终结论的内容

- 义务库条目目前是预审映射雏形，不替代逐条法规条款 verified。
- PTQ-检测报告覆盖矩阵仍基于可抽取文本关键词，表格 PDF、扫描件和复杂跨页表格仍需后续增强。
- 驾驶舱准备度是演示评分，用于排序和沟通风险，不代表正式注册可通过概率。
